BLOCKTOWN

在不規則格網上，蓋一座奶油色小鎮

TL;DR

用瀏覽器即時建造的 3D 小鎮玩具，模仿 Townscaper 風格。屋頂、門窗、欄杆、簷口全部自動生成。純原生 HTML/CSS/JS、沒 build 工具，打開網頁就跑、60fps。

Blocktown 是一個瀏覽器 3D 建築玩具，受 Oskar Stålberg 的 Townscaper 啟發。透過程序生成不規則格網、Loop subdivision 圓潤建築、地基紅色 cornice 與鑄鐵欄杆、3+3 種凹陷感門窗變體、動態水波紋 + foam，整體在 60fps 即時運算。

核心特色

01 · TERRAIN

程序生成地形

第一步：起點。從整齊的方格網開始，用一個圓形 mask 切出小島輪廓——所有不在圓內的格子都丟掉。

第二步：變有機。對整片格網施加波形扭曲，讓行與列像水波一樣彎，產生 Townscaper 風的「流動感」；再對內部頂點加上微抖動，打破過於規則的感覺。

第三步：收斂。跑等邊鬆弛把所有 quad 的邊長磨到差不多；順帶過濾掉因為 mask 切出來的離島碎塊。整段過程邊界頂點全程鎖定不動，確保外輪廓始終是圓島。最終產出 441 個 quad cells。

Why：單純方格太呆板、純隨機會崩成碎片。「先扭曲、再抖動、再等邊化」三步漸進，每一步都退一點規則、多一點有機，是兼顧「不亂」與「不呆」的中間解。

02 · VISUAL

圓潤 + 桶身 + 程序材質

第一步：揉圓。用 Loop subdivision 把硬邊的方塊揉成 puffy 圓潤形狀，像揉麵團一樣把所有銳角都磨成弧線。每個方塊的顏色直接「烘」進頂點，揉圓時也會自然平滑過渡。

第二步：桶身。地基底部往外擴一點，形成 Townscaper 招牌的「碗形地基浮在水面」造型。樓層上的地板獨立一層 mesh，讓簷口能緊貼邊緣。

第三步：材質。屋頂自動鋪上瓦片圓點、牆面加上水平木紋；水面跑動態波紋與泡沫；天空有自然飄移的雲層；最後整張畫面套上電影感色調與柔和暈光。

Why：方塊原本是硬邊立方體，跟 Townscaper 那種「碗形地基浮在水面」的視覺差很遠。揉圓 + 桶身 + 程序材質三層疊，才生出這種「夢幻又溫暖」的整體氛圍。

03 · AUTO-DECO

自動裝飾系統

第一步：屋頂自動長。程式判斷每塊方塊周圍格子的狀況，自動決定屋頂該長成金字塔、屋脊還是平頂；屋脊兩端會自動升起垂直三角牆。

第二步：屋簷自動推。屋頂外緣統一向外突出形成屋簷；遇到 L 形凹角時自動收進來，形成圓滑的過渡曲線，避免穿模或裂縫。

第三步：配件自動裝。地基頂自動包上紅色簷口帶與鑄鐵欄杆；每棟建築外圍再套上一圈水面泡沫光暈。

第四步：門窗自動選。三種門 + 三種窗的變體，每棟建築用自己的編號隨機選一組樣式；外框前置 + 玻璃凹進去一層，從外看像「往牆內看進去」的洞，門窗看起來有深度。

Why：讓玩家為每棟建築決定 30 個細節（屋頂形狀、欄杆、門窗位置、凹角怎麼處理 …）會崩潰。Townscaper 最神奇的就是「程式幫你想好所有細節」，本作從屋頂到門窗全部自動生成，玩家只負責放方塊。

技術堆疊

THREE.JS 0.160 ↗

📦 把 3D 畫面畫到網頁上的工具，就像「網頁版的 Unity」。

WebGL、PCF 軟陰影、ACES Filmic Tonemapping、OrbitControls、PerspectiveCamera。

DELAUNATOR 5 ↗

📦 把一堆「點」連成不重疊的三角形，地圖學常用。

Delaunay 三角剖分演算法。本作改用 square grid + curve distortion + jitter，但保留 utility 給後續實驗。

THREE-SUBDIVIDE ↗

📦 把硬邊方塊「揉圓」成可愛形狀，像揉麵團。

Loop subdivision：每個三角形分成 4 個並 smoothing，把硬邊 block geometry 圓潤成 Townscaper 風 puffy 形狀。

GLSL SHADER ↗

📦 寫給顯卡的迷你程式，告訴它每個像素長什麼樣。本作用它畫天空、水、瓦片。

六套自訂 shader — 天空 FBM 雲 + 漸層球、徑向漸層水面、動態波紋 + foam 飄移雜訊、水線交界 highlight、屋頂瓦片圓點、牆面水平木紋。

POST-PROCESSING ↗

📦 畫完場景後再加一層濾鏡，就像 IG 修圖。本作加了微暖暈光。

EffectComposer + UnrealBloomPass：strength 0.35 / threshold 0.85 微暈光，增加溫暖夢幻感。

ES MODULE + IMPORT MAP ↗

📦 不用打包工具，瀏覽器自己處理 JS 模組。打開 HTML 就能跑。

零 build：純原生 ES Module 從 CDN 載入，靜態 server 就能跑，無 webpack / vite。

generateGrid()

雙層 sine 波 — Curve Distortion

讓行列像水波一樣彎曲，產生 Townscaper 風的 flow 感。雙層相位錯開的 sine 波，振幅 0.35、頻率 0.32，seed 影響相位。

const phaseX = rnd() * Math.PI * 2;
const phaseZ = rnd() * Math.PI * 2;
for (const v of usedVerts) {
  const [x, z] = pts[v];
  // x 位移取決於 z（垂直波）
  const dx = Math.sin(z * CURVE_FREQ + phaseX) * CURVE_AMP
           + Math.sin(z * CURVE_FREQ * 1.73 + phaseX + 1.1) * CURVE_AMP * 0.4;
  // z 位移取決於 x（水平波）
  const dz = Math.cos(x * CURVE_FREQ + phaseZ) * CURVE_AMP
           + Math.cos(x * CURVE_FREQ * 1.73 + phaseZ + 2.3) * CURVE_AMP * 0.4;
  pts[v] = [x + dx, z + dz];
}
computeVertexHeightWeights()

region 內部度 — Auto Roof

對每個頂點掃描相鄰邊：兩側都是同 region 同層 cell 視為 internal edge。fraction = internal / total。0 為屋簷邊界、1 為屋脊內部。

let internalEdges = 0, totalEdges = 0;
for (const cid of vertexToCells.get(vidx)) {
  if (!regionSet.has(cid)) continue;
  const cell = cells[cid];
  const vi = cell.vertIdx.indexOf(vidx);
  // 該頂點觸碰兩條邊：edge[vi] 與 edge[vi-1]
  const edgeIdxs = [vi, (vi - 1 + N) % N];
  for (const ei of edgeIdxs) {
    totalEdges++;
    if (regionSet.has(cell.neighbors[ei])) internalEdges++;
  }
}
// fraction >= 0.5 直接拉到頂 → 共邊端點變屋脊
const w = Math.min(1, internalEdges / totalEdges * 2);
injectBlockShader()

onBeforeCompile — 瓦片圓點

透過 Three.js material 的 onBeforeCompile hook 注入 GLSL，用 world-space derivatives 推 face normal，再給朝上的面加交錯排列的圓點瓦片。

// === 屋頂：交錯排列圓點瓦片 ===
float roofW = smoothstep(0.25, 0.55, wN.y);
if (roofW > 0.0) {
  vec2 t = vWPos.xz * uTileScale;
  // 每隔一行橫向偏移半格，做出磚面般的錯位
  if (mod(floor(t.y), 2.0) > 0.5) t.x += 0.5;
  vec2 local = fract(t) - 0.5;
  float d = length(local);
  float dot = smoothstep(0.44, 0.28, d);
  diffuseColor.rgb *= (1.0 - dot * uTileStrength * roofW);
}

功能亮點

  • 兩階段建造：第一次點空地 = 矮地基（沉到水下 −0.20 延伸到水面 +0.3）；之後 = 一格高樓層 + 屋頂。
  • 屋頂自動形變：依 region 內部度決定頂點抬升，孤立格 → 金字塔、雙格 → gable、大叢集 → 平頂；gable 端再升垂直三角牆。
  • 紅色 cornice 簷口帶：地基頂兩色 cornice（外推 12cm / 內收 4cm）跳過 subdivision 獨立 mesh，Townscaper 風招牌。
  • 鑄鐵欄杆 railing：地基頂沿外圍立柱 + 雙橫桿 InstancedMesh，邊長均分立柱間距。
  • 3 種門 + 3 種窗 variant：拱形 / 矩形雙開 / 單門+雨遮 · 拱窗 / 4-pane 十字框 / 圓窗；每棟用 bid hash 固定一種。
  • 凹陷感門窗：外框前置 + 玻璃在背面 z=0，從外看洞裡是凹進去的面板。
  • 動態水波紋 + foam：徑向漸層水面外加 sin wave + value noise 飄移雜訊 shader，水線交界加 highlight。
  • Hover Ghost + cinematic 選單：3D 半透明預覽呼吸動畫；左側漸層黑 + monospace 選項支援 ↑/↓/Enter 鍵盤導航。

挑戰與解決

CASE 01

兩階段建造的設計演進

問題：最初所有格子第一次點擊都當「樓層」，但矮層樓在水面平行視角看起來像漂浮的方塊，沒有「島」的厚度。

解：抽出第一層為「地基」podium：從水下 WATER_Y − 0.20 延伸到 FOUNDATION_TOP_Y = 0.3，第二次起才是樓層 + 屋頂。地基頂還加紅色 cornice 帶與鑄鐵欄杆強化 Townscaper 視覺。

CASE 02

rebuildAll 順序 bug

問題：原本 rebuildAll 先用 buildingCells 重建 mesh，再 refreshDecorations 更新 buildingCells。第一次種 demo town 時跑進空陣列，沒生出任何 mesh。

解：把 refreshDecorations() 移到 mesh rebuild 之前，先重算分組再用新的 buildingCells 迭代。

CASE 03

屋簷穿模

問題：屋簷外推會造成多種破圖：鄰格較高時插進牆裡；多 cell 排列時邊緣不均勻；ridge 端點縮回成裂縫；gable 牆頂與屋頂 apex 共平面 Z-fight 破圖。

解：四刀並行 — (a) 跨 region 法線累加：所有 boundary vert 統一外推 22cm，連續均勻不縮回；(b) 垂直 gable 牆：牆頂 Y 跟 roofBase 升到 ridge 高度形成三角牆；(c) Z-fight cap：牆頂 Y 限制 apex − 10cm；(d) 點擊判定用 hit 點 Y 而非 normal.y。

CASE 04

稀疏陣列導致 undefined.color

問題：seedDemoTown 用條件機率「跳過某層」造成 blocks 陣列稀疏（blocks[1] undefined 但 blocks[2] 有值），rebuildBuildingMesh 讀到 undefined.color 炸掉。

解：改用 blocks.push(...) 強制連續疊，邏輯改成 nested if 確保上層只在下層存在時才疊。

CASE 05

選單繞鏡頭的「停頓感」

問題：menuOrbitT 寫死 (1/60) 累加 + shadow.autoUpdate=true + 4096² 陰影貼圖，遇 GC 或 frame drop 時鏡頭看起來會停住一拍。

解：三刀並行 — (a) dt-based 累加且 clamp 100ms 上限；(b) shadow.autoUpdate=false 只在場景變動才 flag；(c) mapSize 4096²→2048²。穩定 60fps。

三件最值得驕傲的事

  1. 01

    零建置依賴

    純 ES Module 從 CDN 載入，沒 webpack、沒 vite、沒 npm install。任何靜態 server 就能跑，甚至直接 GitHub Pages 部署。

  2. 02

    整套自動裝飾系統

    玩家不用想屋頂該長什麼樣、門開哪、窗放哪、欄杆怎麼包——程式依 region 內部度自動長屋頂（金字塔／gable／平頂），最長外牆放門、其他放窗（3+3 變體 hash 選樣），地基頂自動沿外圍包 cornice 與鑄鐵欄杆。

  3. 03

    絲滑 60fps

    dt-based 動畫累加、陰影按需更新、bloom 後處理三件事一起跑不卡。用真實量測（fps counter）找到瓶頸再針對性優化。

一路上學到什麼

💡 效能最佳化不是空想出來

原本以為「shader 應該很慢」拍腦袋猜原因，profile 後才發現是 4096² shadow map 每幀重算，跟 shader 沒關係。先量測再下刀，比猜十次都準。

🐛 大多數 bug 都來自順序假設

rebuildAll 順序錯、seedDemoTown 稀疏陣列、屋簷外推方向，都是「我以為 A 會比 B 先發生」這類隱性假設。把順序寫成註解是最便宜的防線。

🎨 UX 細節比功能更耗時

Hover ghost、平滑追焦鏡頭、cinematic 選單、鍵盤導航、PROJECT INFO 這頁本身花的時間 是核心建造邏輯的 2 倍。但這些才是讓人記得住的部分。
